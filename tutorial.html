Konsep Sistem

Apa yang Bisa Anda Jalankan dengan iMersTeleShip?

iMersTeleShip dirancang untuk menjual produk digital, membership, kelas online, template, lisensi, dan akses private langsung dari Telegram.

▣

Toko Digital Telegram

Produk tampil di bot, pembeli klik katalog, pilih produk, lalu checkout tanpa harus diarahkan ke sistem rumit.

◇

Database Google Sheets

Produk, member, order, lisensi, dan pengaturan bot dikelola lewat Google Sheets yang mudah diedit.

▤

Pembayaran Manual

Mendukung transfer bank dan QRIS manual. Admin bisa approve atau reject pesanan dari Telegram.

↯

White-Label Branding

Nama bot, logo, pesan otomatis, copywriting, dan halaman panduan bisa disesuaikan dengan brand Anda sendiri.

Bagian 1 — Persiapan Awal

Siapkan Kunci Utama Sebelum Instalasi

Sebelum menjalankan sistem, Anda perlu membuat bot Telegram dan mengambil Admin ID. Dua data ini akan dipakai agar bot bisa mengirim katalog, menerima order, dan mengirim notifikasi ke admin.

1. Dapatkan Token Bot

Buka @BotFather di Telegram, ketik /newbot, ikuti instruksi, lalu copy API Token yang diberikan.

2. Dapatkan Admin ID

Buka @userinfobot di Telegram untuk melihat ID angka akun Anda. ID ini dipakai untuk menerima notifikasi order.

3. Gunakan Akun Gmail Pribadi

Disarankan memakai akun @gmail.com, bukan email kantor/kampus, supaya izin Apps Script dan deployment lebih lancar.

Bagian 2 — Instalasi & Deployment

Ikuti Urutan Ini Agar Bot Aktif Tanpa Error

Jangan lompat langkah. Urutan yang benar adalah salin template, setup database, deploy web app, lalu aktifkan webhook dari dashboard.

01

Salin Database Template

Buka link Google Sheets template, lalu klik File → Buat Salinan. Setelah berhasil disalin, gunakan file salinan itu sebagai database utama.

  • Jangan edit template asli.
  • Pastikan sheet bisa dibuka dengan akun Gmail Anda.
  • Berikan nama file sesuai brand Anda.
02

Setup Database Awal

Di Google Sheets, klik menu 🚀 iMersTeleship App → ⚙️ Setup Database Awal.

  • Membuat sheet Settings.
  • Membuat sheet Products.
  • Membuat sheet Orders, Members, dan Licenses.
03

Masukkan Data Dummy

Klik 📝 Masukkan Data Dummy agar ada contoh produk. Ini membantu Anda mengetes katalog tanpa input manual dari nol.

  • Cek format harga.
  • Cek status produk Active/Aktif.
  • Cek tombol beli dan detail produk.
04

Buka Apps Script

Klik Ekstensi → Apps Script. Pastikan semua file script sudah ada, termasuk Code.gs, Telegram.gs, Payment.gs, Admin.gs, dan Dashboard.html jika tersedia.

  • Klik Save sebelum deploy.
  • Jalankan authorization jika diminta.
  • Pastikan tidak ada error merah di editor.
05

Deploy Web App

Klik Deploy → New Deployment → Web App, lalu gunakan konfigurasi berikut:

Execute as: Me / Saya Who has access: Anyone / Siapa saja URL yang dipakai: link berakhiran /exec
06

Aktifkan Bot

Buka Dashboard Admin dari menu Sheets, masukkan Token Bot, Admin ID, dan URL Web App /exec, lalu klik Simpan & Aktifkan.

  • Bot token dari BotFather.
  • Admin ID dari userinfobot.
  • Web App URL dari hasil deploy.
Bagian 3 — Bedah Menu Add-On

Fungsi Menu iMersTeleShip di Google Sheets

Menu ini menjadi pusat kontrol utama untuk dashboard, database, testing, diagnosa, dan emergency reset bot.

Menu Fungsi Kapan Dipakai
💻 Buka Dashboard Admin Membuka panel kontrol untuk mengatur API, produk, harga, link akses, sales page, dan teks otomatis. Saat ingin setup bot, tambah produk, atau edit pengaturan.
🤖 Uji Coba: Ping Bot Mengirim pesan test ke admin untuk memastikan bot bisa mengirim pesan dari Apps Script. Saat ingin cek token dan Admin ID valid atau tidak.
🛍️ Uji Coba: Katalog Produk Mengirim katalog produk ke Telegram admin untuk melihat tampilan produk. Saat selesai input produk atau edit katalog.
📡 Diagnosa: Cek Status Webhook Melihat status webhook, pending update, dan error dari server Telegram. Saat bot tidak merespon, delay, atau muncul error 302/403.
⚙️ Setup Database Awal Membuat struktur tabel sistem secara otomatis di Google Sheets. Saat pertama kali instalasi atau sheet belum lengkap.
📝 Masukkan Data Dummy Mengisi contoh produk, setting, dan data testing agar sistem langsung bisa dicoba. Saat ingin demo cepat tanpa input manual.
🛑 Hentikan Spam Bot Mematikan webhook bot sementara. Versi terbaik harus memakai drop_pending_updates agar antrean lama ikut dibuang. Saat bot looping, spam welcome, atau membalas pesan lama.
🔄 Reset & Bersihkan Antrean Bot Menghapus webhook lama, membuang pending updates, lalu memasang webhook ulang secara bersih. Saat pending_update_count tinggi atau bot memproses antrean lama.
Bagian 4 — Manajemen Toko

Tambah Produk, Kelola Order, dan Broadcast Member

Setelah bot aktif, Anda bisa mulai memasukkan produk digital dan menjalankan alur transaksi dari Telegram.

  • Tambah Produk: buka Dashboard Admin → tab Produk → isi nama, tipe, harga, deskripsi, link sales page, link file, dan status.
  • Kelola Pesanan: ketik /admin di bot Telegram, lalu pilih order pending untuk approve atau reject.
  • Broadcast: ketik /admin → Broadcast, lalu kirim pesan promosi ke seluruh member bot.
  • Deep Link Produk: gunakan link checkout otomatis untuk dipasang di landing page, CTA, bio, atau halaman produk.

Troubleshooting Penting

Kalau Bot Tidak Merespon, Cek Bagian Ini Dulu

Mayoritas masalah biasanya berasal dari deploy belum versi terbaru, webhook salah URL, token salah, Admin ID salah, atau antrean Telegram menumpuk.

Bot tidak membalas ketika klik Produk Kami
Pastikan webhook aktif, deployment sudah versi terbaru, dan sheet Products memiliki produk dengan status Active atau Aktif. Gunakan menu Uji Coba: Katalog Produk untuk mengetes katalog langsung ke admin.
Bot looping mengirim welcome terus
Ini biasanya karena pending update Telegram menumpuk. Gunakan menu Reset & Bersihkan Antrean Bot. Fungsi reset idealnya memakai drop_pending_updates=true agar antrian lama dibuang dan tidak diproses ulang.
Muncul error 302 Found di webhook
Gunakan URL deployment yang berakhiran /exec, bukan URL editor Apps Script. Setelah deploy ulang, simpan kembali konfigurasi di Dashboard Admin agar webhook mengarah ke URL yang benar.
Muncul error 403 Forbidden
Biasanya bot diblokir oleh user, Admin ID salah, atau izin Apps Script belum benar. Jalankan Ping Bot untuk memastikan bot bisa mengirim pesan ke admin.
Setelah edit source code, bot masih pakai versi lama
Apps Script tidak otomatis memakai kode terbaru jika deployment belum diperbarui. Klik Deploy → Manage deployments → Edit → Version: New version → Deploy, lalu simpan ulang config webhook.
Token bot terlihat atau bocor
Segera buka @BotFather, pilih bot Anda, lalu gunakan fitur revoke token. Setelah mendapat token baru, masukkan ulang di Dashboard Admin dan aktifkan webhook lagi.
Checklist Final

Sebelum Sistem Dipakai Jualan, Pastikan Ini Sudah Aman

  • Bot sudah dibuat dari @BotFather dan token tersimpan di Dashboard.
  • Admin ID sudah benar dan admin menerima pesan test Ping Bot.
  • Web App sudah dideploy dengan akses Anyone.
  • Webhook sudah aktif ke URL /exec terbaru.
  • Sheet Products berisi minimal 1 produk dengan status Active/Aktif.
  • Produk memiliki link file/akses yang benar.
  • Instruksi bank atau QRIS sudah diisi.
  • Menu /admin bisa dibuka oleh admin.
  • Alur checkout, upload bukti, approve order, dan buka akses sudah dites.
